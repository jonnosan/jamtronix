# Ableton + JTX — setup guide

How to build an Ableton Live Set that plays songs generated by the
Composer. The Composer always emits the **same fixed 9-voice palette**
plus a 3-voice utility cluster, so this set is constructed once and
reused for every song you generate. Only the per-voice musical
content changes as you sweep mood / texture / motion in the GUI.

Each voice has a stylistic role — what kind of *sound* it expects.
The recipe below describes that character per channel. The exact
instrument is your call: drop whatever fits the role into the bundled
`JtxCCRack` and Map the Macros to its params.

## What jtx sends to Live

* MIDI on `IAC Driver Bus 1` (default — change in the setup editor if
  your setup needs a different port).
* **12 MIDI channels** per the bundled `setups/ableton.jtx-setup`:
  9 musical voices on channels 1–8 (with the drum kit fanned out
  across 9, 10, 11) plus 3 utility voices (filter on 12, root + chord
  pitch followers on 15 and 16).
* **CC traffic per voice** for filter cutoff, resonance, glide,
  glide-on, modulation, and three user-routable CCs (102/103/104).
  The bundled `JtxCCRack.adg` Instrument Rack pre-wires its eight
  Macros to those CCs; you only ever Map Macro → instrument param.

## CC → Macro routing

The same routing applies to every musical track. `JtxCCRack` ships
with this wiring already in place.

| jtx function | MIDI CC | Rack Macro |
|---|---|---|
| `cutoff` | 74 | Macro 1 — Cutoff |
| `resonance` | 71 | Macro 2 — Resonance |
| `glide` | 5 | Macro 3 — Glide |
| `glide_on` | 65 | Macro 4 — Port |
| `detune` / mod | 1 | Macro 5 — Mod |
| (user-routable) | 102 | Macro 6 — Aux 1 |
| (user-routable) | 103 | Macro 7 — Aux 2 |
| (user-routable) | 104 | Macro 8 — Aux 3 |

CC 102–104 sit in MIDI's "undefined" range and are kept free for
arbitrary user routing — LFO targets, FX sends, anything you point a
`cc_lfo` or `step_cc` modulator voice at.

## One-time setup

### Install the Rack preset

Copy `daw_templates/JtxCCRack.adg` into your Ableton User Library so
Live shows it in the browser:

```
~/Music/Ableton/User Library/Presets/Instruments/Instrument Rack/
```

Drag the file in Finder, then restart Live (or refresh the browser).
`JtxCCRack` should appear under **User Library → Presets → Instruments
→ Instrument Rack**.

### Configure Live's MIDI input

`Live → Preferences → Link / Tempo / MIDI`:

1. Set **IAC Driver Bus 1**'s **Track** column to **On** so tracks can
   listen on it. (Remote and Sync stay off unless a specific track
   needs them.)

## Track layout

Twelve tracks, one per voice the Composer emits.

| jtx voice | MIDI ch | Type | Role |
|---|---|---|---|
| `drumkit` | 9 / 10 / 11 | drum rack | Four-on-the-floor club kit |
| `bass` | 1 | mono synth | Dual-character bass (Reese ↔ acid) |
| `sub` | 2 | mono synth | Pure sub-bass weight |
| `lead` | 3 | mono synth | Front-of-mix solo voice |
| `pad` | 4 | poly synth | Lush atmospheric glue |
| `chord` | 5 | poly synth | Mid-band chord body |
| `arp` | 6 | mono synth | Bright repeating figure |
| `stabs` | 7 | poly synth | Short percussive chord hits |
| `fx` | 8 | mono synth | Noise / atmospheres / risers |
| `filter` | 12 | (no instrument) | Global cutoff modulator — see below |
| `root_ref` | 15 | (optional) | Root-pitch follower for hardware |
| `chord_ref` | 16 | (optional) | Chord-pitch follower for hardware |

The nine musical tracks each carry one `JtxCCRack` with an instrument
inside, mapped per voice character. The filter track is pure CC — no
instrument. The two `*_ref` tracks are only useful if you're chaining
external gear and want a pitch reference; otherwise omit them.

## Per-voice sound recipes

These describe *character*, not specific products. Pick instruments
that nail the character; the routing is identical regardless.

### `drumkit` — channels 9, 10, 11

A drum rack receiving a kit split across three channels so each group
can run its own bus / FX chain if you want:

| Group | MIDI ch | Notes (GM-ish) |
|---|---|---|
| Kick | 9 | 36 (C1) |
| Snare + hi-hats + clap | 10 | 38 / 42 / 46 / 39 |
| Toms + clave + shaker + perc | 11 | 45 / 75 / 82 / 64 |

The kick should be the centrepiece — punchy, low-mid weight, with a
short tail you'd sidechain the bass + pad around. Snare wants a
forward club character (not orchestral). Closed hi-hat should be
short and tight; open hat with enough decay to swirl across the
backbeat. Clap is for offbeats. The perc group (toms / clave / shaker
/ perc cell) covers the smaller hit-shaped sounds that thicken the
groove.

The simplest workable layout is one drum rack on a single track that
listens to all three channels (Live lets a track listen to multiple
channels via dropdown), with kit cells at the notes above. If you
want separate bus / compression / reverb sends for kick vs hats vs
perc, give each group its own track and listen to one channel.

### `bass` — channel 1

The most stylistically loaded voice. The Composer switches its
algorithm based on motion:

* **Low motion**: Reese bass — dark, sustained, detuned saw stack
  with a slow LFO on the detune (CC1 / Macro 5). Foundation that sits
  *behind* the kick.
* **Higher motion**: acid bass — 303-style mono, slide between
  successive notes (CC5 + CC65 / Macros 3 + 4), squelchy resonant
  filter (CC74 + CC71 / Macros 1 + 2), short envelope per note.

You want **one instrument** that can convincingly do both — a mono
synth with a beefy lowpass + resonance, glide on demand, and enough
saw / sub character to growl at low motion. Cutoff and resonance are
the two macros that earn their keep here; without them, neither the
Reese growl nor the acid squelch will land.

### `sub` — channel 2

Pure sub-bass weight. Sine-shape or near-sine, no high content,
nothing fancy. Its job is to make the song feel large on a club
system; on laptop speakers you'll barely hear it. Avoid any drive,
filter movement, or stereo width — keep it dead centre, mono, deep.

The composer often gives `sub` a sustained drone pattern that holds
the root across the bar, layered under whatever the `bass` voice is
doing. The two work as a pair: `bass` does the line; `sub` does the
weight.

### `lead` — channel 3

The "star" voice. Front-of-mix, expressive, designed to carry a
melodic line. At low motion the Composer feeds it a phrase-style
pattern (short motifs with rests); at higher motion it gets a
running melodic line. Either way the instrument should have:

* A clear, present tone (not buried under wash).
* Some kind of articulation under your control — filter envelope,
  amp envelope, vibrato (Macro 5 / CC1 is a natural pick for
  modulation depth).
* A comfortable range from mid-register to upper.

Think "the synth you'd notice if it dropped out" — that's the lead.

### `pad` — channel 4

The atmospheric glue. Slow attack, long release, lush stereo image,
sits behind everything. The composer holds chords across whole bars
on this voice — its job is space, not motion. Reach for a poly synth
with wide unison, plenty of stereo movement, a slow filter envelope
that opens across the note, and big reverb / chorus to taste.

Density-wise the pad only activates at mid-to-high texture, and at
low motion in particular — high motion biases the τ threshold *up*
for the pad (it's the wrong voice for high-energy psy-style output).
When it does come in it should feel like the room got bigger.

### `chord` — channel 5

A mid-band polyphonic voice that fills the harmonic body between the
bass and the lead. The composer's algorithm choice swings with motion:

* **Low motion**: sustained chord — held across the bar like a
  smaller version of the pad.
* **Higher motion**: chord stab — short, percussive chord with a
  fast amp envelope, often with the filter accent under the
  Composer's control.

Pick a poly synth with a snappy amp envelope, a good filter for
movement, and reasonable mid-range presence — Rhodes-y electric
pianos, classic poly-synth strings, plucky poly leads all fit
depending on the song. The instrument should be able to swing
between "held" and "stabbed" without sounding like two different
patches.

### `arp` — channel 6

A mono voice running a repeating figure at a chosen subdivision
(eighths through sixteenths depending on motion). The character to
reach for: bright, percussive, short — a plucky pluck or a hollow
saw with a fast envelope. Reverb and delay are encouraged; this voice
benefits from spatial smear because the notes themselves are short
and dense.

At low motion it leans melodic-arp (longer notes, slower subdivision).
At high motion it becomes the busy little engine running underneath
the lead — fast subdivisions, almost trance-gate feel. The same
instrument can serve both if its envelope tightens nicely under the
Composer's pattern.

### `stabs` — channel 7

Short, percussive polyphonic chord hits. Distinct from `chord` in
that `stabs` is *always* short — never sustained — and tends to fire
on off-beats or syncopated positions for groove. Think classic house
piano stabs, organ stabs, brass stabs, or a chord pluck with a hard
gate. The amp envelope should be fast attack + fast release with no
sustain.

Texture-wise this voice activates very early (τ = 0.05) — it's
present in nearly every song, even sparse ones. Make sure whatever
sits here is something you don't mind hearing in every output.

### `fx` — channel 8

Noise, atmospheres, and risers. Two distinct modes by motion:

* **Low / mid motion**: `step_cc` mode — the voice doesn't play
  notes; it just streams stepped CC values (CC1 / Macro 5 — "noise"
  by default). Use this to drive a *noise level* parameter, *aux
  send level*, or any param you want stepped rhythmically. The
  instrument can be a noise generator with mod-controlled output, or
  the channel can hold an FX rack whose first device exposes its
  send level on Macro 5.
* **High motion**: `noise_riser` mode — the voice plays a sustained
  note with a rising filter envelope and noise content, the classic
  pre-drop riser. The instrument here should be a noise synth (or a
  pad with a heavily noise-flavoured patch) whose filter cutoff
  (CC74 / Macro 1) sweeps the riser.

The pragmatic choice: a noise/atmosphere synth that's musical at all
filter positions. Or a sampler loaded with a noise wash + a sweepable
filter.

## Utility tracks

### `filter` — channel 12 (no instrument)

This track has no sound. It only exists so the Composer can broadcast
a **global filter cutoff modulator** (CC74) that breathes across the
song — a slow stepped sweep that opens and closes across bars, with
depth and subdivision driven by motion.

You don't put an instrument on this track. Instead, set it up to
forward its CC74 to the musical tracks that should breathe under the
global sweep. The simplest pattern:

1. Track listens to `IAC Driver Bus 1`, channel 12.
2. Monitor `In` (so CC traffic flows even when the track isn't
   armed).
3. Send the track's MIDI out to each musical track you want under
   the global sweep, OR keep it as a CC-receive-only track and route
   each musical track's Macro 1 (Cutoff) to the same CC74 input
   directly.

In practice the easiest workflow is: every musical track's
`JtxCCRack` is already listening for CC74 on its own channel, *and*
the `filter` voice's stream is a separate "global" CC74 on channel
12. If you want the global sweep to affect a given voice, route its
Macro 1 to receive both — Live's MIDI mapping lets a single Macro
listen to multiple CC sources. Pick the voices you want under the
breathing sweep (bass, pad, chord typically); leave the others
unaffected so the kick and lead don't pump.

### `root_ref` and `chord_ref` — channels 15, 16

Two follower voices that re-emit the root note of the `bass` voice
and the bottom note of the `chord` voice on their own channels.
Useful only if you've got a downstream device (an external synth, a
hardware sequencer, a Eurorack rig) that needs to latch onto the
song's current harmonic state without parsing the entire bass /
chord MIDI stream.

For a pure-software Live set these tracks can be omitted entirely
without affecting playback. The `setups/ableton.jtx-setup` still
ships them so the JSON matches the Composer's output verbatim.

## Per-track wiring procedure

For each musical track (drumkit, bass, sub, lead, pad, chord, arp,
stabs, fx):

1. Add a **MIDI track**, rename it after the voice.
2. **MIDI From**: `IAC Driver Bus 1`. **Channel**: the channel from
   the table above (drumkit listens to 9, 10, and 11 — multi-channel
   listen, or split across 3 tracks).
3. **Monitor**: `In`.
4. **MIDI To**: `No Output` (unless feeding hardware downstream).
5. Drag **`JtxCCRack`** from your User Library onto the track. The
   eight Macros come up already wired to the CCs above.
6. Drag your chosen instrument inside the Rack. The Rack UI shows a
   "Drop an Instrument here" zone.
7. Click Live's **Map** button (top-right). Click each Macro, then
   click the instrument param it should drive. Click **Map** again
   to exit Map mode.

The CC → Macro plumbing is baked into the Rack; you only ever map
Macro → instrument.

## Swap workflow

When you decide a different instrument fits a voice better:

1. Inside the Rack, drag in the new instrument; drag the old one
   out.
2. Click **Map**.
3. Click each Macro in turn → click the new instrument's matching
   param.
4. Click **Map** again.

Done. The jtx side is untouched, no MIDI Learn, no audition. The
CC → Macro mappings inside the Rack persist across all such swaps.

## Ableton Link

jtx supports Ableton Link as a clock source: Setup editor → General
tab → Clock mode → "Ableton Link". Requires the optional
`LinkPython-extern` dependency:

```
pip install 'jamtronix[link]'
```

When Link is active, jtx tempo follows the Link session; start/stop
participates in the shared transport. No further setup — Link
auto-peers on the local network.
